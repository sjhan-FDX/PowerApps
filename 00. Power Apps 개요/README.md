# 00. Power Apps 개요

## 교육 목표

Power Apps의 개념과 특징을 이해하고, Canvas App과 Model-driven App의 차이점을 학습합니다. 또한 Power Platform의 구성 요소와 Power Apps 개발 환경을 이해하여 이후 실습을 위한 기초를 다집니다.

---

# 1. Microsoft Power Platform 소개

Microsoft Power Platform은 기업의 업무를 디지털화하고 자동화하기 위한 Low-Code 플랫폼입니다.

Power Platform은 다음과 같은 서비스로 구성됩니다.

| 서비스                | 설명                        |
| ------------------ | ------------------------- |
| **Power Apps**     | 업무용 애플리케이션 개발             |
| **Power Automate** | 업무 프로세스 자동화               |
| **Power BI**       | 데이터 분석 및 시각화              |
| **Copilot Studio** | AI Agent 및 Copilot 개발     |
| **Dataverse**      | Power Platform 공통 데이터 저장소 |

---

# 2. Power Apps란?

Power Apps는 프로그래밍 경험이 많지 않은 사용자도 웹 및 모바일에서 사용할 수 있는 업무용 애플리케이션을 빠르게 개발할 수 있도록 지원하는 Microsoft의 Low-Code 개발 플랫폼입니다.

### 주요 특징

* Low-Code / No-Code 개발
* 다양한 데이터 원본 연결
* Microsoft 365 서비스 연동
* 웹 및 모바일 동시 지원
* 빠른 업무용 앱 개발

---

# 3. Power Apps 활용 사례

Power Apps는 다양한 업무에 활용할 수 있습니다.

* 상품 관리
* 재고 관리
* 휴가 신청
* 출장 신청
* 점검(Check List)
* 설비 관리
* 고객 관리
* 영업 관리

---

# 4. Power Apps 종류

## Canvas App

사용자가 원하는 형태로 화면(UI)을 자유롭게 설계하는 앱입니다.

**특징**

* 자유로운 화면 구성
* Power Fx 사용
* Excel, SharePoint, Dataverse 등 다양한 데이터 연결
* 모바일 앱 개발에 적합

---

## Model-driven App

Dataverse를 기반으로 데이터를 중심으로 구성되는 앱입니다.

**특징**

* Dataverse 기반
* 데이터 중심 개발
* 화면 자동 생성
* 권한 및 보안 관리에 강점
* CRM, ERP와 같은 업무 시스템에 적합

---

# 5. Canvas App vs Model-driven App

| 구분    | Canvas App                     | Model-driven App  |
| ----- | ------------------------------ | ----------------- |
| 개발 방식 | 화면(UI) 중심                      | 데이터 중심            |
| UI 구성 | 자유롭게 설계                        | 자동 생성             |
| 데이터   | Excel, SharePoint, Dataverse 등 | Dataverse         |
| 활용 분야 | 업무용 앱, 모바일 앱                   | CRM, 영업 관리, 자산 관리 |

> **본 교육에서는 Canvas App을 활용하여 Excel 기반 상품 관리 앱을 제작합니다.**
